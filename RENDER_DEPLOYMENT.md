## স্টেপ ২: প্রয়োজনীয় পরিবেশ ভেরিয়েবলগুলো প্রস্তুত করুন

আপনার Render ড্যাশবোর্ডে নিম্নলিখিত এনভায়রনমেন্ট ভেরিয়েবলগুলো সেট করুন:

### ডাটাবেস
- `DATABASE_URL` - Render-এ আপনার PostgreSQL ডাটাবেসের সংযোগ স্ট্রিং

### টেলিগ্রাম (ঐচ্ছিক - যোগাযোগ ফর্মের জন্য)
- `TELEGRAM_BOT_TOKEN` - আপনার টেলিগ্রাম বট টোকেন
- `TELEGRAM_CHAT_ID` - আপনার টেলিগ্রাম চ্যাট আইডি

## স্টেপ ৪: Render-এ ডিপ্লয় করুন

### অপশন A: render.yaml ব্যবহার করে (সুপারিশকৃত)
1. Render ড্যাশবোর্ডে "New" বাটন ক্লিক করুন
2. "Blueprint" নির্বাচন করুন
3. আপনার GitHub রিপোজিটরি সংযুক্ত করুন
4. `render.yaml` স্বয়ংক্রিয়ভাবে সনাক্ত হবে

### অপশন B: ম্যানুয়াল সেটআপ
1. "New" > "Web Service" নির্বাচন করুন
2. আপনার GitHub রিপোজিটরি সংযুক্ত করুন
3. নিম্নোক্ত সেটিংস ব্যবহার করুন:
   - **Runtime**: Node
   - **Build Command**: `npm install && npm run build`
   - **Start Command**: `NODE_ENV=production npm run start`
   - **Environment Variables**: উপরে তালিকাভুক্ত ভেরিয়েবলগুলো যোগ করুন

## স্টেপ ৫: ডাটাবেস সেটআপ (যদি স্বয়ংক্রিয়ভাবে না হয়)

1. Render ড্যাশবোর্ডে "New" > "PostgreSQL" নির্বাচন করুন
2. ডাটাবেস নাম দিন (যেমন: `portfolio-db`)
3. ডাটাবেস তৈরি হওয়ার পর, সংযোগ স্ট্রিং কপি করুন
4. আপনার Web Service-এ `DATABASE_URL` এনভায়রনমেন্ট ভেরিয়েবল সেট করুন

## স্টেপ ৬: মাইগ্রেশন চালান

একবার ডিপ্লয় সম্পূর্ণ হলে, ডাটাবেস টেবিলগুলো স্বয়ংক্রিয়ভাবে তৈরি হবে যখন অ্যাপ্লিকেশন প্রথম শুরু হয়।

## ট্রাবলশুটিং

### "relation 'skills' does not exist" ত্রুটি
এর মানে ডাটাবেস টেবিলগুলো এখনও তৈরি হয়নি। অ্যাপ্লিকেশনটি স্বয়ংক্রিয়ভাবে তৈরি করবে, কিন্তু যদি সমস্যা হয়:

1. Render ড্যাশবোর্ডে আপনার Service নির্বাচন করুন
2. "Shell" ট্যাবে যান
3. এই কমান্ডটি চালান:
   ```bash
   npx drizzle-kit push
   ```

### DATABASE_URL সেট না থাকলে
নিশ্চিত করুন যে আপনি:
1. PostgreSQL সেবা তৈরি করেছেন
2. সংযোগ স্ট্রিং সঠিকভাবে কপি করেছেন
3. Web Service-এ `DATABASE_URL` এনভায়রনমেন্ট ভেরিয়েবল সেট করেছেন

## আপনার অ্যাপ্লিকেশন চেক করুন

ডিপ্লয় সম্পূর্ণ হওয়ার পর, আপনার Render URL-এ ভিজিট করুন এবং অ্যাপ্লিকেশনটি কাজ করছে কিনা তা যাচাই করুন।
